Requisitos Funcionais

Integração com Dispositivos

  1. Objetivo: Garantir a coleta e processamento preciso dos dados das atividades físicas realizadas pelos usuários.
  2. Atores:
  3. Usuário
  4. Dispositivos de medição (ex: relógios inteligentes)
  5. Pré-condições:
  6. O dispositivo de medição deve estar emparelhado com o aplicativo.
  7. O usuário deve ter permitido o acesso aos dados do dispositivo.
  8. Trigger:
  9. O início de uma atividade física (natação, corrida, ciclismo).
  10. Fluxo Principal:
  11. O usuário inicia uma atividade física no aplicativo ou no dispositivo.
  12. O dispositivo coleta os dados de desempenho (ex: batimentos cardíacos, distância, tempo).
  13. O aplicativo processa e exibe os dados em tempo real para o usuário.
  14. Ao fim da atividade, o aplicativo armazena o histórico da sessão.
  15. Fluxos Alternativos:
  16. Fluxo Opcionais: O usuário pode visualizar uma comparação com atividades anteriores.
  17. Fluxo Variante: Diferentes dispositivos podem enviar dados adicionais, como níveis de oxigênio ou sono.
  18. Fluxo de Exceção: Se o dispositivo perder a conexão, o aplicativo tenta reconectar ou notifica o usuário para reiniciar a conexão.
  19. Pós-condições:
  20. O histórico de desempenho da atividade é salvo no perfil do usuário.
  21. Regras:
  22. O aplicativo deve verificar a compatibilidade do dispositivo e seu estado de bateria.

Precisão no GPS

  1. Objetivo: Melhorar a exatidão dos dados de localização e percurso.
  2. Atores:
  3. Usuário
  4. Sistema de GPS
  5. Pré-condições:
  6. O GPS do dispositivo deve estar ativado e funcional.
  7. Trigger:
  8. O início de uma atividade que requer rastreamento (ex: corrida, ciclismo).
  9. Fluxo Principal:
  10. O usuário inicia o rastreamento da atividade.
  11. O GPS captura dados de localização e percurso com precisão.
  12. O aplicativo exibe o trajeto em um mapa em tempo real.
  13. Ao final da atividade, o percurso é salvo.
  14. Fluxos Alternativos:
  15. Fluxo Variante: O usuário pode revisar o percurso em diferentes níveis de zoom.
  16. Fluxo de Exceção: Caso a precisão do GPS caia, o aplicativo tenta recalibrar ou notifica o usuário.
  17. Pós-condições:
  18. O trajeto completo e preciso é salvo no histórico.
  19. Regras:
  20. O aplicativo deve garantir o mínimo de desvio possível na precisão do GPS, considerando margens de erro aceitáveis.

Requisitos Esperados

Funcionalidades de Rede Social

  1. Objetivo: Promover a interação social e a motivação entre os usuários.
  2. Atores:
  3. Usuário
  4. Outros membros de grupos ou assessorias esportivas
  5. Pré-condições:
  6. O usuário deve estar inscrito em um grupo ou assessoria esportiva.
  7. Trigger:
  8. Um membro do grupo conclui uma atividade ou compartilha uma atualização.
  9. Fluxo Principal:
  10. O usuário visualiza as atividades dos outros membros do grupo.
  11. O usuário pode reagir ou comentar nas atividades.
  12. As reações/comentários são salvos e exibidos em tempo real.
  13. Fluxos Alternativos:
  14. Fluxo Opcional: O usuário pode ativar notificações para ser informado quando outros membros completarem atividades.
  15. Pós-condições:
  16. As interações sociais (comentários/reações) são registradas no perfil do usuário.
  17. Regras:
  18. O aplicativo deve garantir que apenas membros autorizados visualizem e interajam com atividades.

Atualizações e Novidades por Email

  1. Objetivo: Manter os usuários informados sobre melhorias e novas funcionalidades do app.
  2. Atores:
  3. Usuário
  4. Sistema de envio de email
  5. Pré-condições:
  6. O usuário deve estar inscrito para receber emails.
  7. Trigger:
  8. O aplicativo ou seus desenvolvedores lançam uma atualização ou notícia relevante.
  9. Fluxo Principal:
  10. O usuário opta por receber atualizações por email.
  11. O sistema envia notificações periódicas de novidades.
  12. O usuário recebe o email e pode acessar informações adicionais.
  13. Pós-condições:
  14. O usuário está informado sobre as últimas novidades do aplicativo.

Requisitos Excitantes

Estreitamento de Relações entre Membros

  1. Objetivo: Fomentar uma comunidade engajada e conectada.
  2. Atores:
  3. Usuário
  4. Membros de grupos esportivos
  5. Pré-condições:
  6. O usuário deve fazer parte de um grupo.
  7. Trigger:
  8. Criação de fóruns, desafios ou competições amistosas.
  9. Fluxo Principal:
  10. O grupo organiza desafios ou fóruns de discussão.
  11. O usuário participa, comentando ou competindo.
  12. O aplicativo registra o progresso do desafio ou fórum.
  13. Fluxos Alternativos:
  14. Fluxo Opcionais: O usuário pode criar seus próprios desafios dentro de grupos menores.
  15. Pós-condições:
  16. O grupo se fortalece com as interações e desafios realizados.

Personalização Avançada

  1. Objetivo: Oferecer uma experiência rica e adaptada às preferências individuais.
  2. Atores:
  3. Usuário
  4. Pré-condições:
  5. O usuário deve configurar suas preferências no aplicativo.
  6. Trigger:
  7. O usuário acessa a área de personalização.
  8. Fluxo Principal:
  9. O usuário ajusta o perfil e as metas no aplicativo.
  10. O aplicativo oferece recomendações e feedback personalizados.
  11. Fluxos Alternativos:
  12. Fluxo Variante: As metas podem ser ajustadas dinamicamente com base no desempenho do usuário.
  13. Pós-condições:
  14. As metas e feedbacks são atualizados e adaptados à evolução do usuário.

Requisitos Não Funcionais

Desempenho

  1. Objetivo: Garantir eficiência e ausência de atrasos significativos.
  2. Atores:
  3. Usuário
  4. Servidor do aplicativo
  5. Pré-condições:
  6. O aplicativo deve ser executado em dispositivos compatíveis.
  7. Trigger:
  8. A utilização de grandes volumes de dados ou funcionalidades complexas.
  9. Fluxo Principal:
  10. O usuário utiliza o aplicativo sem interrupções ou atrasos, mesmo com grandes volumes de dados.
  11. Pós-condições:
  12. O aplicativo mantém um desempenho estável.

Segurança e Privacidade

  1. Objetivo: Proteger informações sensíveis e garantir a privacidade dos usuários.
  2. Atores:
  3. Usuário
  4. Sistema de segurança do aplicativo
  5. Pré-condições:
  6. O usuário deve ter um login seguro.
  7. Trigger:
  8. O usuário acessa ou envia dados sensíveis.
  9. Fluxo Principal:
  10. O sistema valida as credenciais do usuário.
  11. O aplicativo criptografa e protege os dados armazenados.
  12. Pós-condições:
  13. Os dados do usuário estão seguros e protegidos.
